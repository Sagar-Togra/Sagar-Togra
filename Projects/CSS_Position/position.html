CSS Position Property

Syntax  :  static | relative | fixed | absolute | sticky;

Lets know about each value one by one.

Static

  • It is the default setting.
  • It isn't affected by the top, bottom, left and rigth properties.
This div has position set static.

Relative

  • When an element position is set to relative, it can be moved relative to its normal position by using top, bottom, left and right property.
  • It leaves a gap from where it was moved.
This div has position set to relative with top property of 50px and left property of 50px.

Fixed

  • Element with position set to fixed is positioned relative to the viewport.
  • The scroll has no effect on this. It will stay where it is.
  • This doesn't leave any gap.
You can see this div in the top right of the viewport.
This div has position set to Fixed. It is in the bottom rigth of viewport.

Absolute

  • Element with position set to absolute is positioned relative to the neared positioned parent element.
  • In absence of a positioned parent, element is positioned relative to the body by default.
  • It is removed from the normal flow adn can overlap

This is the parent div

This div has position set to absolute with bottom property of 10px and right property of 10px.

As stated earlier it can overlap lets see for ourself.

This is the parent div

This div has position set to absolute with bottom property of -30px and right property of 10px.

This line is being overlapped.

Sticky

  • Element with position set to sticky is positioned based on user's scroll position.
  • It is relative when it in the viewport when outsitd of viewport it becomes fixed.
  • Note
    • It does't work in Internet explorer
    • In safari you need to add "-webkit-" prefix.
this div has position set to sticky with top 20px and left 10px.
